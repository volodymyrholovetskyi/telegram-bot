1.Запуск програми:
Перед стартом необхідно впровадити дані оператора у файл application.properties,
а також дані необхідня для зєднання з бекенодм телеграма (name & token).
Для швидкого запуску програми достатньо встаноити середовище виконання Docker Engine, відкрити cmd,
перейти в папку проекту та виконати команду: docker-compose up -d після цієї команди буде створено два контейнера,
один на основі образу нашого додатку та інший на основі Postgresql, після чого наш додаток готовий до використання.
Клонування проекту:
git clone https://github.com/volodymyrholovetskyi/telegram-bot.git

2. Технології
java 17
Spring Boot 3.2.0
MySql 8.0 (docker-compose для можливості запуску БД без необхідності інсталяції)
Liquibase - бібліотека для контролю і керування схемою бази даних
Telegram-Bot-Api 6.8.0

3. Команди:
/start - перша команда повертає привітання користувача в чаті
/help - список доступних команд
/register - реєстрація користувача, після натискання кнопки "Send contact" Bot зберігає в БД наступні дані:
сatId, firstName, lastName, phoneNumber.
/date_of_visit - дата і час у відповідному форматі

4. Логіка виконання програми:
Щоб забронювати квиток, користувач повинен бути зареєстрований. Після реєстрації користувач має можливість
бронювати квитки шляхом введення дати візиту, дата перевіряється, чи вона не введена заднім числом і має відповідний формат.
Кожні 5 хвилин активується Планувальник і збирає всі квитки зі статусом BOOKED після
чого генерує білет, який зберігається в корені проекту в папці tickets і надсилається користувачеві у форматі
FirstNameLastNameYearMonthDayHourMinute.txt іпісля чого також відправляється повідомлення операторові,
після всіх цих операцій статус квитка змінюється на CONFIRMED.

5. Інше:
Приклад створеного білету можна переглянути в папці tickets в корні проекту.
